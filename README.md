# .github

Shared GitHub templates and workflows for HersRD repos.

## Issue Templates

| Template | Labels | Description |
|----------|--------|-------------|
| [Bug Report](.github/ISSUE_TEMPLATE/bug.yml) | `bug` | Report a bug to help us identify and fix issues. Includes fields for bug description, steps to reproduce, expected behavior, actual behavior, screenshots, and additional context. |
| [Spike](.github/ISSUE_TEMPLATE/spike.yml) | `spike`, `documentation` | Research and document a new feature, technology, or implementation approach. Includes fields for objective, research type, background, options to explore, evaluation criteria, expected output, time box, and story points. |
| [Story](.github/ISSUE_TEMPLATE/story.yml) | `feature` | Create a user story issue to describe a feature or enhancement from the user's perspective. Includes fields for a user story narrative, acceptance criteria, additional context, and story point sizing. |

### Configuration

- **Blank issues are disabled** ([config.yml](.github/ISSUE_TEMPLATE/config.yml)) — contributors must use one of the templates above when opening an issue.
